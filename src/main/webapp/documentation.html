Javadocs

Libgdx comes with full Javadocs. You can read them online, or check your local copy in the release or nightly build you downloaded.

If you created your projects with the Gdx Setup, the source Jar files will be automatically added to your projects so you can benefit from them inside your IDE.

Wiki

We are in the process of building out our development guide Wiki. You can contribute directly on Github, the wiki is open for everyone!

Getting started

  1. Setup your development environment (Eclipse, Intellij IDEA, NetBeans)
  2. Create a libgdx project
  3. Import, run, debug and package your project with
    • Eclipse
    • Intellij IDEA
    • NetBeans
    • Commandline
  4. Read the Wiki!
  5. Join the Forums and IRC
  6. Follow our Blog and Twitter (@badlogicgames, @libgdx)

Demos

Libgdx comes with a few demos that you can play around with to get a feeling for how to do things. Note that many of these demos have been created during game jams and may not necessarily reflect best practices.

Setup your development environment, then simple clone the repositories on Github (or click on the "Download ZIP" button of a project's site) and import the project into your favorite IDE as a Gradle project

Cuboc - a simple platformer with a twist

Gdx Invaders - a 3D space invaders clone

Pax Britannica - a one button RTS game

Super Jumper - a Doodle jump clone with programmer's art

Vector Pinball - a Box2D-based pinball game

Tests

Another source of inspiration our our test examples which we use to verify libgdx functionality on different platforms. You can find these in the libgdx repository. To play around with these tests you need to:

  • Setup Eclipse (sorry, we'll be back with Gradle soon)
  • Read up on how to work from source, import and run the tests
